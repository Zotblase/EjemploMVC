Explicación del MVC
Cómo está organizado nuestro programa en el MVC: El MVC inicia desde la interfaz gráfica que pide los valores a ingresar en los diferentes campos. Con estos datos ingresados la vista llamará al controlador para comprobar que sean correctos. El controlador comprobará los datos y devolverá un true o false. Una vez que todos los datos hayan sido ingresados y comprobados, la vista, llamará al modelo para guardar los datos. Por último la vista tiene la posibilidad de volver a llamar al modelo para consultar los datos de los usuarios y mostrarlos por pantalla.
Modelo: en el modelo guardaremos los datos de nuestros usuarios. Para esto, crearemos una Arraylist que guardará objetos de la clase User. Ya que estabamos le metimos un Singleton a esta clase modelo, ya que queremos que los usuarios se creen en una y sólo una Arraylist. Además, esta clase tiene el método agregarUsuario para agregar usuarios al Arraylist. 
Vista: Creación de la interfaz con sus botones e inputs. La interfaz funciona de forma que, después de poner los valores en todos los campos, se aprieta el botón de enviar. Con esto, la Vista se comunicará con el Controlador para comprobar que los datos sean correctos. En caso de que alguno no esté ingresado correctamente la interfaz genera una nueva ventana con detalles del error. Por último, la vista es capaz de mostrar a todos los usuarios registrados con el botón "Mostrar Usuarios"
Controlador: el controlador provee los métodos para comprobar que los datos sean correctos. Tenemos el método comprobarRegistro que chequea si el usuario ya está registrado en el sistema utilizando el correo. El método comprobarEdad que comprueba si el valor es mayor a 0. el comprobarEmail que comprueba que el correo sea correcto (que tenga un nombre, un arroba, un dominio y un punto com). El comprobarContrasenia que comprueba que la contraseña sea correcta
